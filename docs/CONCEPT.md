# MusicRoots — Музыкальное генеалогическое древо

> Крупнейшая база **подтверждённых** музыкальных влияний. Не энциклопедия, а карта
> творческой преемственности: кто на кого повлиял, через какие эпохи и поколения.

---

## 1. Идея и ключевой принцип

MusicRoots визуализирует не факты о музыкантах, а **связи влияния** между ними.
Главный объект платформы — **древо влияний**, а страницы артистов — вспомогательный
справочный материал.

Каждая связь — это **направленное ребро** между двумя узлами (артист или группа) и
означает подтверждённое творческое влияние. Цепочки строятся вида:

```
Артист → Артист → Артист
Элвис Пресли → Артур Крудап → Сон Хаус
Metallica → Motörhead → Jimi Hendrix
```

### Направление стрелки

Стрелка `A → B` читается как **«A испытал влияние B»** (B — корень, источник, «предок»).
Это естественное генеалогическое направление: от потомка к предку, от нового к корням.

- `Metallica → Motörhead` — Metallica черпали из Motörhead.
- `Motörhead → Jimi Hendrix` — Motörhead черпали из Хендрикса.

Таким образом, двигаясь по стрелкам, пользователь спускается **к корням** музыки;
двигаясь против стрелок — поднимается к **наследникам** и более поздним поколениям.

### Главный инвариант данных

> **Связь существует, только если у неё есть хотя бы один достоверный источник.**

Без источника связь не публикуется — она остаётся в статусе предложения на модерации.

---

## 2. Достоверность источников

Допустимые типы источников (поле `source.type`):

| Тип            | Описание                                          |
| -------------- | ------------------------------------------------- |
| `interview`    | Интервью самих музыкантов                          |
| `autobiography`| Автобиографии и мемуары                            |
| `encyclopedia` | Музыкальные энциклопедии (AllMusic, Grove и т.п.) |
| `documentary`  | Документальные фильмы и материалы                  |
| `official`     | Официальные публикации, пресс-релизы, сайты       |
| `other`        | Иные авторитетные источники                        |

Для каждого источника хранится: тип, цитата/описание, ссылка (если есть), издание/автор,
дата. У одной связи может быть несколько источников — чем их больше, тем выше «уровень
доверия» связи (см. §6, индикатор confidence).

## 3. Архитектура данных

Структура влияний — это **направленный граф** (в общем случае с возможными циклами:
взаимное влияние), а не строгое дерево. «Древо» — это пользовательская проекция графа
относительно выбранного артиста (см. §5).

### Сущности

```
Artist (артист или группа)
  id, slug, name, type (artist|band), bio,
  active_from, active_to, country, image_url, tags[]

Influence (ребро графа: artist испытал влияние influenced_by)
  id, artist_id        → кто испытал влияние (потомок)
      influenced_by_id → источник влияния (корень)
      description       → в чём именно выразилось влияние
      status (approved|pending|rejected)

Source (подтверждение конкретной связи)
  id, influence_id, type, citation, url, author, year

Contribution (пользовательское предложение на модерацию)
  id, kind (new_influence|new_source|error_report|correction),
      payload (JSON), submitter, status, moderator_note,
      created_at, reviewed_at
```

### Почему граф, а не дерево

- У одного артиста **несколько источников влияния** → несколько входящих рёбер.
- Один артист влияет на многих → несколько исходящих рёбер.
- Возможны общие предки и схождения цепочек (например, и блюз, и кантри ведут к общим
  корням) → структура с разветвлением и слиянием (DAG), иногда с циклами.

Хранилище — реляционное (SQLite в прототипе, PostgreSQL в продакшене). Граф удобно
хранить рёбрами и обходить через рекурсивные CTE/BFS на стороне API.

## 4. Структура сайта и навигация

```
/                     Главная: «войти в древо» — поиск + точки входа
/explore              Полноэкранное древо влияний (центральный экран платформы)
/explore?focus=<slug> Древо, центрированное на конкретном артисте
/artist/<slug>        Мини-страница артиста (справка + локальное древо)
/contribute           Формы участия сообщества
/moderation           Очередь модерации (для модераторов)
/about                О проекте, методология, правила источников
```

Навигация устроена вокруг **непрерывного исследования**: из любой точки (поиск, страница
артиста, узел графа) пользователь одним кликом перецентрирует древо на нужном артисте и
продолжает движение по цепочкам.

---

## 5. Древо влияний — центральный элемент

### Модель отображения

Граф проецируется в **иерархический (генеалогический) лейаут** относительно
сфокусированного артиста:

- Сфокусированный артист — в центре.
- **Вниз/влево** по стрелкам — корни влияния (предки) на нескольких уровнях глубины.
- **Вверх/вправо** против стрелок — наследники (на кого повлиял), кого вдохновил артист.
- Уровни (поколения) визуально разнесены — видно движение сквозь эпохи.

### Взаимодействие

- **Pan & zoom** по холсту.
- **Клик по узлу** → перецентрировать древо на этом артисте (ядро навигации).
- **Двойной клик / кнопка** → открыть мини-страницу артиста.
- **Наведение на ребро** → всплывающая подсказка с описанием и источниками связи.
- **Регулятор глубины** (1–3 уровня в каждую сторону), чтобы не перегружать экран.
- **Фильтры**: только корни / только наследники / по эпохе / по уровню доверия.
- **Хлебные крошки пути** — пройденная пользователем цепочка артистов.

### Технически

Прототип использует force-directed/иерархический рендер на SVG (D3-подобный лейаут по
глубине BFS). В продакшене — Canvas/WebGL для больших подграфов и серверная подрезка
подграфа вокруг фокуса (запрос «дай N уровней соседей»).

## 6. Уровень доверия связи (confidence)

Чтобы отражать достоверность, у связи вычисляется индикатор:

- число и **тип** источников (первичные — интервью/мемуары — весомее вторичных);
- наличие прямой цитаты музыканта;
- подтверждение несколькими независимыми источниками.

Индикатор показывается цветом/толщиной ребра и значком на связи. Это поощряет
сообщество усиливать слабые связи новыми источниками, а не плодить недоказанные.

**Правило публикации.** Уровни `high` и `medium` публикуются; `low` — нет. Слабые
связи не отображаются, не участвуют в построении древа и не рисуются на графе (серых
стрелок нет). Confidence вычисляется по весам источников: `interview`/`autobiography` =
3, `documentary`/`encyclopedia`/`official` = 2, `other` = 1; сумма ≥ 5 → `high`, ≥ 3 →
`medium`, иначе `low`. Чтобы поднять каноничную, но слабо подтверждённую связь до
публикуемой, к ней добавляют второй независимый достоверный источник.

## 7. Участие сообщества и модерация

Пользователь может через `/contribute`:

1. **Предложить новую связь** (артист A ← влияние B) с обязательным источником.
2. **Добавить источник** к существующей связи.
3. **Сообщить об ошибке** в данных.
4. **Предложить исправление** существующих данных.

Все предложения попадают в `contributions` со статусом `pending` и **не публикуются**
до решения модератора. В `/moderation` модератор видит очередь, контекст предложения и
кнопки **Одобрить / Отклонить** с комментарием. При одобрении предложение применяется к
основным данным (создаётся/обновляется `Influence`/`Source`).

### Антиспам и качество (план продакшена)

- Репутация участников, лимиты на число заявок.
- Дубликат-детекция предложенных связей.
- Требование источника для любой связи (жёсткий инвариант).
- История изменений и возможность отката.

## 8. Пользовательские сценарии

**Исследователь.** Вводит «Nirvana» → видит древо: корни (Pixies, The Stooges, Black
Flag…) и наследники. Кликает Pixies → перецентрирование → спускается к корням панка и
далее. Прослеживает цепочку через поколения.

**Любопытствующий.** С главной выбирает случайную точку входа, гуляет по связям, читает
всплывающие источники, сохраняет интересный путь (хлебные крошки).

**Контрибьютор.** Замечает отсутствующую связь, открывает `/contribute`, заполняет форму
с цитатой из интервью и ссылкой. Заявка уходит на модерацию.

**Модератор.** В `/moderation` проверяет заявку, открывает источник, одобряет — связь
появляется в древе у всех пользователей.

## 9. Дорожная карта

- **MVP (этот прототип):** граф-древо, страницы артистов, поиск, формы участия,
  очередь модерации, сид реальных подтверждённых связей.
- **v1:** аутентификация и роли, серверная подрезка подграфа, полнотекстовый поиск,
  индикатор confidence, импорт из открытых датасетов с верификацией.
- **v2:** репутация участников, версии данных и откаты, рекомендации путей,
  тематические подборки цепочек, публичный API и встраиваемый виджет древа.
