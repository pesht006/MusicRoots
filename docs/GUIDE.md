## 1. Структура проекта

```
MusicRoots/
├─ web/                     ← Сайт (то, что видит пользователь). React + Vite
│  ├─ src/
│  │  ├─ data/seed.json     ← СНИМОК ДАННЫХ для онлайн-демо (генерируется автоматически)
│  │  ├─ api.ts             ← типы данных + выбор источника (сервер или браузер)
│  │  ├─ staticApi.ts       ← логика данных в браузере (для GitHub Pages)
│  │  ├─ components/
│  │  │  ├─ InfluenceGraph.tsx  ← само древо влияний (граф)
│  │  │  ├─ Avatar.tsx          ← фото артиста (с запасным «инициалом»)
│  │  │  └─ SearchBox.tsx       ← поиск
│  │  └─ pages/
│  │     ├─ HomePage.tsx        ← главная
│  │     ├─ ExplorePage.tsx     ← экран древа + боковая панель
│  │     ├─ ArtistPage.tsx      ← страница артиста
│  │     ├─ ContributePage.tsx  ← формы предложений (для пользователей)
│  │     └─ ModerationPage.tsx  ← очередь модерации (для админа)
│  └─ scripts/gen-data.mjs  ← собирает web/src/data/seed.json из данных сервера
│
├─ server/                  ← Бэкенд (полная версия с базой). Node + SQLite
│  └─ src/
│     ├─ seed-data.js       ← ★ ГЛАВНЫЙ ФАЙЛ ДАННЫХ: артисты, связи, источники, фото
│     ├─ db.js              ← структура базы (таблицы)
│     ├─ seed.js            ← наполняет базу из seed-data.js
│     ├─ graph.js           ← построение древа (подграфа)
│     └─ index.js           ← API (эндпоинты)
│
├─ docs/                    ← Документация (CONCEPT.md — концепция, GUIDE.md — этот файл)
└─ .github/workflows/deploy.yml  ← автопубликация сайта на GitHub Pages
```

### Главное, что нужно знать

- **Единственный файл, который вы редактируете для контента, — это
  [`server/src/seed-data.js`](../server/src/seed-data.js).** В нём задаются артисты,
  связи влияния, источники и фотографии.
- После правки этого файла нужно **пересобрать снимок данных** для онлайн-версии
  командой `npm --prefix web run gen:data` (или это сделает CI при публикации —
  команда вызывается автоматически во время сборки).
- Онлайн-демо (GitHub Pages) работает **без сервера**: данные «вшиты» в сайт из
  `web/src/data/seed.json`, а предложения/модерация хранятся в браузере
  пользователя (`localStorage`). Полная версия с настоящей базой и общей для всех
  модерацией запускается локально (см. README).

## 2. Инструкция для пользователя

### Что такое древо влияний

Сайт показывает, **кто на кого повлиял** в музыке. Стрелка `A → B` читается как
«**A испытал влияние B**» (B — это «корень», источник). Двигаясь по стрелкам, вы
спускаетесь к истокам музыки; двигаясь против — поднимаетесь к более поздним
музыкантам-наследникам.

### Навигация

1. **Главная** — введите имя артиста в поиск или нажмите «Открыть древо».
2. **Экран «Древо»** (`/explore`):
   - **Центр (фиолетовый)** — выбранный артист.
   - **Слева (оранжевые)** — его корни влияния (на кого он опирался).
   - **Справа (синие)** — наследники (на кого он повлиял).
   - **Колесо мыши** — масштаб; **перетаскивание** — двигать полотно.
   - **Клик по узлу** — перецентрировать древо на этом артисте.
   - **Двойной клик по узлу** — открыть полную страницу артиста.
   - **Наведение на стрелку** — всплывает описание связи и число источников.
   - **Ползунок «Глубина»** — сколько уровней показывать (1–4) в каждую сторону.
   - **«Путь»** сверху — цепочка артистов, по которой вы шли (хлебные крошки).
3. **Боковая панель** (справа на экране древа): фото и биография артиста, списки
   «Корни влияния» и «Повлиял на». У каждой связи есть ссылки **«источник ↗»** и
   **«Wikipedia ↗»**.
4. **Страница артиста** (`/artist/...`): фото, годы активности, страна, полный список
   подтверждённых влияний и наследников — **с цитатами источников и ссылками на них**.

### Уровень доверия

Цвет стрелки и метка показывают, насколько связь подтверждена:
🟢 `high` — высокий, 🟡 `medium` — средний, ⚪ `low` — требует подтверждения.
Он зависит от числа и типа источников (интервью и мемуары весомее).

### Как предложить изменение (любой пользователь)

Раздел **«Участвовать»** (`/contribute`) — четыре типа заявок:

- **Новая связь** — указываете, кто на кого повлиял, **и обязательно источник**.
- **Новый источник** — добавить подтверждение к существующей связи.
- **Сообщить об ошибке** — что-то в данных неверно.
- **Исправление** — предложить правку.

Все заявки уходят на модерацию и не публикуются сразу.

> В онлайн-демо ваши заявки сохраняются только в вашем браузере, и одобрить их можно
> в разделе «Модерация» прямо здесь же (это демонстрация процесса).

## 3. Инструкция для администратора

Есть два способа управлять данными.

- **Способ A (через сайт, для отдельных заявок)** — раздел «Модерация».
- **Способ B (через файл данных, для массового/надёжного редактирования)** — правка
  `server/src/seed-data.js`. Это основной способ наполнять базу.

### 3.1. Модерация заявок (через сайт)

1. Откройте раздел **«Модерация»** (`/moderation`).
2. Вкладки: **На рассмотрении / Одобрено / Отклонено**.
3. По каждой заявке видны её тип, содержимое и источник. Кнопки:
   - **Одобрить** — заявка применяется к данным (новая связь/источник появляются в древе).
   - **Отклонить** — с комментарием модератора.

> В полной версии (с сервером) одобрение меняет общую базу для всех. В онлайн-демо
> изменения сохраняются в браузере текущего пользователя.

### 3.2. Как добавить нового музыканта

Откройте [`server/src/seed-data.js`](../server/src/seed-data.js) и добавьте объект в
массив `artists`:

```js
{
  slug: "nine-inch-nails",        // уникальный id латиницей через дефис (для ссылок)
  name: "Nine Inch Nails",        // отображаемое имя
  type: "band",                   // "band" (группа) или "artist" (исполнитель)
  country: "США",
  active_from: 1988,
  active_to: null,                 // null = по настоящее время
  tags: ["industrial-rock"],
  bio: "Индастриал-проект Трента Резнора.",
},
```

`slug` должен быть **уникальным** — по нему строятся адреса страниц
(`/artist/nine-inch-nails`) и связи.

### 3.3. Как добавить новую связь влияния

В том же файле добавьте элемент в массив `influences` с помощью помощника `E(...)`:

```js
E("nine-inch-nails", "the-stooges",
  "Резнор называл The Stooges одним из ключевых влияний.",
  [
    S("interview", "Trent Reznor в интервью назвал Iggy/The Stooges формирующим влиянием.",
      "Trent Reznor", 1994, "https://example.com/interview"),
  ]),
```

Сигнатуры помощников:

```js
E(артист, источник_влияния, описание, [массив_источников])
//  ↑ кто испытал влияние   ↑ «корень»

S(тип, цитата, автор, год, ссылка)
```

- **`E(A, B, ...)`** означает «**A испытал влияние B**» (стрелка `A → B`).
  Оба значения — это `slug` уже существующих артистов.
- **Каждой связи нужен хотя бы один `S(...)`** (источник) — это жёсткое правило проекта.
- Допустимые **типы источника** (`type`): `interview`, `autobiography`,
  `encyclopedia`, `documentary`, `official`, `other`.
- В `S(...)` поля `автор`, `год`, `ссылка` необязательны, но **ссылка** — это как раз
  то, что показывается пользователю как «источник ↗». Если хотите указать только
  ссылку без года, передайте `null` вместо года:
  `S("encyclopedia", "AllMusic: ...", "AllMusic", null, "https://...")`.

### 3.4. Как добавить фотографию музыканта

Фотографии заданы в `server/src/seed-data.js` в объекте `imageMap` (ключ — `slug`,
значение — прямая ссылка на изображение):

```js
const imageMap = {
  "nine-inch-nails": "https://upload.wikimedia.org/.../NIN.jpg",
  // ...
};
```

Рекомендации:

- Используйте **прямые ссылки на картинку** (заканчиваются на `.jpg`/`.png`). Хорошо
  подходят файлы Wikimedia Commons или формат
  `https://commons.wikimedia.org/wiki/Special:FilePath/Имя_файла.jpg`.
- Если фото не задано или ссылка не открылась — автоматически показывается «плитка с
  инициалом», ничего не ломается.
- Ссылка на статью Wikipedia задаётся автоматически из `wikiTitles` (тоже в этом
  файле) — её можно поправить там же.

### 3.5. Применить изменения и опубликовать

1. **Пересоберите снимок данных** для онлайн-версии:

```bash
npm --prefix web run gen:data
```

   (и при работе с полной версией — обновите базу: `npm --prefix server run seed`)

2. **Закоммитьте и запушьте** изменения:

```bash
git add server/src/seed-data.js web/src/data/seed.json
git commit -m "data: добавлен Nine Inch Nails и связь с The Stooges"
git push
```

3. После пуша в ветку `main` (или текущую рабочую ветку) GitHub Actions
   автоматически соберёт сайт и обновит онлайн-демо на GitHub Pages за ~1 минуту.

> Если вы только что включили Pages: **Settings → Pages → Source: «Deploy from a
> branch» → Branch `gh-pages` / `(root)`**. Дальше всё обновляется само.

### 3.6. Запуск полной версии локально (по желанию)

```bash
npm run setup   # установка зависимостей + наполнение базы
npm run dev     # API :4000 + сайт :5173
```

В этом режиме работает настоящая база (SQLite) и общая очередь модерации через API.
