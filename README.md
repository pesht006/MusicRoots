# MusicRoots — музыкальное генеалогическое древо

Платформа визуализации **подтверждённых творческих влияний** между музыкантами и
группами. Это не энциклопедия: центральный элемент — **интерактивное древо влияний**,
по которому можно начать с любого артиста и проследить цепочку связей сквозь эпохи и
поколения.

```
Элвис Пресли → Артур Крудап → Сон Хаус
Metallica → Motörhead → Jimi Hendrix
```

Стрелка `A → B` читается как **«A испытал влияние B»** (B — корень/источник). Каждая
связь публикуется только при наличии достоверного источника.

> Полная концепция, архитектура данных, логика навигации и пользовательские сценарии —
> в [`docs/CONCEPT.md`](docs/CONCEPT.md).

## Что внутри

| Слой        | Стек                              | Назначение |
| ----------- | --------------------------------- | ---------- |
| `web/`      | React + TypeScript + Vite         | UI, интерактивное древо (SVG, pan/zoom), страницы артистов, формы участия и модерация |
| `server/`   | Node + Express + better-sqlite3   | API влияний, граф-подграфы, источники, очередь модерации |
| `docs/`     | Markdown                          | Концепция и архитектура |

### Ключевые возможности

- **Древо влияний** (`/explore`) — граф с фокусом на артисте: слева корни (предки),
  справа наследники; pan/zoom, регулятор глубины, подсказки с источниками на рёбрах,
  навигация по клику и хлебные крошки пройденного пути.
- **Мини-страницы артистов** (`/artist/:slug`) — биография, годы активности,
  подтверждённые влияния и наследники со списком источников и уровнем доверия.
- **Участие сообщества** (`/contribute`) — предложить связь, добавить источник,
  сообщить об ошибке, предложить исправление. Новая связь требует источника.
- **Модерация** (`/moderation`) — очередь предложений с одобрением/отклонением;
  одобренные изменения сразу попадают в данные.

## Демо (GitHub Pages)

Сайт автоматически публикуется на GitHub Pages через workflow
[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml):
`https://<owner>.github.io/MusicRoots/`.

Поскольку GitHub Pages раздаёт только статику, для демо собирается **браузерная
версия** (`VITE_STATIC=true`): данные влияний загружаются прямо в браузере из снимка
сида ([`web/src/data/seed.json`](web/src/data/seed.json)), а очередь модерации
сохраняется в `localStorage`. Полноценная версия с API и SQLite используется при
локальной разработке (см. ниже).

> Если Pages ещё не включён, откройте **Settings → Pages → Build and deployment →
> Source: GitHub Actions** и перезапустите workflow (вкладка **Actions**). Workflow
> также пытается включить Pages автоматически.

## Запуск (полная версия с API)

Требуется Node.js 18+.

```bash
# установка зависимостей и наполнение базы реальными связями
npm run setup

# запуск API (:4000) и веб-приложения (:5173) вместе
npm run dev
```

Затем откройте <http://localhost:5173>.

Сервисы можно запускать и по отдельности:

```bash
npm run server   # API на :4000
npm run web      # фронтенд на :5173 (проксирует /api на :4000)
npm run seed     # пересоздать и наполнить базу
npm test         # проверки целостности данных
```

База — SQLite-файл `server/data.sqlite`, создаётся командой `seed` (в git не хранится).

## Модель данных (кратко)

- **artists** — артист или группа (slug, имя, тип, био, годы, страна, теги).
- **influences** — ребро `artist_id` ← `influenced_by_id` (кто испытал влияние / источник),
  с описанием и статусом модерации.
- **sources** — подтверждение конкретной связи (тип, цитата, автор, год, ссылка).
- **contributions** — пользовательские предложения на модерацию (4 типа).

Структура влияний — **направленный граф** (с разветвлением и слиянием цепочек), а
«древо» — это его проекция относительно выбранного артиста на N уровней в обе стороны.
Подробнее — в [`docs/CONCEPT.md`](docs/CONCEPT.md).

## API

| Метод | Путь | Назначение |
| ----- | ---- | ---------- |
| `GET`  | `/api/stats` | счётчики |
| `GET`  | `/api/artists?search=` | поиск артистов |
| `GET`  | `/api/artists/:slug` | артист + корни/наследники + источники |
| `GET`  | `/api/graph?focus=:slug&depth=` | подграф для древа |
| `POST` | `/api/contributions` | отправить предложение |
| `GET`  | `/api/contributions?status=` | очередь модерации |
| `POST` | `/api/contributions/:id/approve` · `/reject` | решение модератора |
